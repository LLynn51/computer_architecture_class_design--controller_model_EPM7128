## 指标记录

### 指令占用时钟周期数统计

| 指令 | 操作码 (IR) | 周期数 | 执行流程 |
|------|:----------:|:------:|----------|
| **ADD** | `0001` | **2** | W1(取指) → W2(ALU加法,写回寄存器,更新C/Z标志) |
| **SUB** | `0010` | **2** | W1(取指) → W2(ALU减法,写回寄存器,更新C/Z标志) |
| **AND** | `0011` | **2** | W1(取指) → W2(ALU逻辑与,写回寄存器,更新Z标志) |
| **INC** | `0100` | **2** | W1(取指) → W2(ALU加1,写回寄存器,更新C/Z标志) |
| **LD**  | `0101` | **3** | W1(取指) → W2(计算目的地址→LAR) → W3(读内存→写回DR) |
| **ST**  | `0110` | **3** | W1(取指) → W2(计算目的地址→LAR) → W3(写内存MEMW=1) |
| **JC**  | `0111` | **2** | W1(取指) → W2(若C=1则PCADD=1,否则空操作) |
| **JZ**  | `1000` | **2** | W1(取指) → W2(若Z=1则PCADD=1,否则空操作) |
| **JMP** | `1001` | **2** | W1(取指) → W2(ABUS直通→LPC,无条件跳转) |
| **STP** | `1110` | **2** | W1(取指) → W2(STOP=1,停止运行) |

其中：

- **2 周期指令**（8 条）：ADD、SUB、AND、INC、JC、JZ、JMP、STP —— 算术/逻辑/跳转类
- **3 周期指令**（2 条）：LD、ST —— 访存类，需要额外的 W3 阶段完成内存读写


| 指标 | 含义 | 公式 | 顺序指令 | 流水线 |
| --- | --- | --- | --- | --- |
| Fmax | 控制器可工作的最大时钟频率 | 查编译报告 `*.ran.rpt` 得到 | 76.92 MHz |  |
| Tclk | 控制器可工作的最小时钟周期| 1 / Fmax |  |  |
| CPI | 每条指令平均需要多少个 T3 周期 |  参考 Hennessy & Patterson 书中对 SPECint 程序的统计得出 | 2.34 |  |
| IPS | 每秒执行指令数  | Fmax / CPI |  |
| Throughput | 吞吐率 | 顺序机约等于 IPS |  |
| Latency | 单条指令从取指到完成需要多少周期 |  |  |

备注：
 
1. Fmax 的来源是编译报告*.ran.rpt 中的 "Clock "T3" has Internal fmax of ……" ；
2. Hennessy & Patterson 书中统计了 SPECint 程序中各种类型指令的占比：

| 指令类型 | 典型占比 | 对应`controller.v`指令 |
| --- | --- | --- |
| 算术/逻辑运算 | ~45% | ADD, SUB, AND, INC (2周期) |
| Load | ~22% | LD (3周期) |
| Store | ~12% | ST (3周期) |
| 条件分支 | ~20% | JC, JZ (2周期) |
| 无条件跳转 | ~1% | JMP (2周期) |

代入计算：$$CPI \approx 0.45×2 + 0.22×3 + 0.12×3 + 0.20×2 + 0.01×2 = 2.34$$
